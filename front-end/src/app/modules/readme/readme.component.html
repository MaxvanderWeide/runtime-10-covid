Runtime dashboard

This dashboard presents COVID-19 information for policy makers. While previous COVID-19 research has identified sets of factors such as average demographic age, hot weather, Bacillus Calmette–Guérin (BCG) vaccine and malaria treatment, these factors have been grouped and are hard to distinguish from each other. This dashboard intends to focus on the comparison of policy trends in different climate conditions, while including different demographic factors, to help policy makers understand the impact of policy interventions through the application of the CRISP-DM method.

Data sources

The dashboard uses multiple data sources. OurWorldInData is used as time series data that is not dynamically fetched. This dashboard saves a local copy of the entire data every day. This dataset is extensive, but does not include data on government interventions. Therefore, this dashboard uses Oxford's Government Response Tracker datasets which are merged into one large time series set that is ready to use for descriptive analysis in the dashboard and is used for the machine learning tasks. Live data on the recent amount of cases and deaths is dynamically fetched from several sources, varying for each country. The data sources per country are:
  • Norway is fetched directly from the official government open data repositories.
  • Netherlands is fetched directly from the official government open data repositories.
  • Spain this live data is retrieved from a free-to-use API via covid19api.com.
  • Brazil this live data is retrieved from a free-to-use API via covid19api.com.

Docker

The dashboard is created using Angular for its front-end, Flask for the API and a crontab that fetches data and creates models. The front-end and API are hosted in docker containers. Docker Container is a standardized unit which can be created on the fly to deploy a particular application or environment. This dashboard is separated into three different docker containers build opun three different docker images. To find more information about how the dashboard is hosted/run go to the dashboard github.
